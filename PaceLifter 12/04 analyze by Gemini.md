# PaceLifter 프로젝트 분석 (by Gemini, 2025-12-04)

## 1. 프로젝트 개요
이 프로젝트는 **Flutter** 프레임워크와 **Dart** 언어를 기반으로 개발된 크로스플랫폼 애플리케이션입니다. 프로젝트명은 **PaceLifter**로 보입니다. `android`, `ios`, `web`, `macos`, `linux`, `windows` 폴더가 모두 존재하는 것으로 보아, 다양한 플랫폼을 타겟으로 하고 있습니다.

애플리케이션의 주된 기능은 건강 및 피트니스 데이터, 특히 운동 기록을 분석하는 것으로 추정됩니다. 주요 기능으로는 건강 데이터 가져오기, 파싱, 시각화 등이 포함될 것으로 보입니다.

## 2. 핵심 기술 스택
- **프레임워크**: [Flutter](https://flutter.dev/)
- **프로그래밍 언어**: [Dart](https://dart.dev/)
- **패키지 매니저**: [Pub](https://pub.dev/) (`pubspec.yaml` 파일을 통해 관리)

## 3. 디렉토리 구조 및 주요 파일 분석

- **`lib/`**: 애플리케이션의 핵심 Dart 소스 코드가 위치하는 메인 디렉토리입니다.
    - **`main.dart`**: Flutter 애플리케이션의 시작점(Entry Point)입니다.
    - **`models/`**: 데이터 모델을 정의하는 파일들이 있습니다.
        - `health_record.dart`: 개별 건강 기록 데이터 구조를 정의할 것으로 예상됩니다.
        - `health_workout.dart`: 운동 세션 데이터 구조를 정의할 것으로 예상됩니다.
    - **`services/`**: 비즈니스 로직 및 서비스를 담당하는 파일들이 있습니다.
        - `health_xml_parser.dart`: Apple 건강 앱의 `export.xml`과 같은 XML 형식의 건강 데이터를 파싱하는 기능이 구현되어 있을 가능성이 높습니다.
    - **`screens/`**: UI 화면을 구성하는 파일들이 있습니다.
        - `health_import_screen.dart`: 건강 데이터를 가져오는(import) UI를 담당하는 화면입니다.
    - **`widgets/`**: 재사용 가능한 UI 컴포넌트(위젯)들이 위치합니다.
    - **`providers/`, `utils/`, `constants/`**: 각각 상태 관리, 유틸리티 함수, 상수 등을 관리하기 위한 표준적인 디렉토리 구조입니다.

- **`pubspec.yaml`**: 프로젝트의 메타데이터와 외부 라이브러리 의존성을 관리하는 설정 파일입니다. 이 파일을 분석하면 상태 관리, UI, 외부 API 연동 등에 어떤 패키지를 사용했는지 정확히 알 수 있습니다.

- **`android/` & `ios/`**: 각각 안드로이드(Gradle, Kotlin/Java)와 iOS(Xcode, Swift/Objective-C) 네이티브 프로젝트 파일이 위치합니다.
    - **`ios/Podfile`**: iOS의 의존성 관리 파일(CocoaPods)입니다. `GoogleMaps` 관련 Pod이 포함된 것으로 보아, iOS 버전에서 구글 맵을 사용하는 기능이 있음을 알 수 있습니다.

- **`web/`, `macos/`, `linux/`, `windows/`**: 웹 및 데스크톱 플랫폼에서 애플리케이션을 실행하기 위한 설정 파일과 진입점들이 포함되어 있습니다.

## 4. 추론된 기능

1.  **건강 데이터 가져오기**: 사용자는 `health_import_screen.dart`를 통해 XML 파일 형태의 건강 데이터를 앱으로 가져올 수 있습니다.
2.  **데이터 파싱**: `health_xml_parser.dart`의 로직을 통해 XML 파일을 분석하여 `HealthRecord`, `HealthWorkout` 같은 의미 있는 데이터 객체로 변환합니다.
3.  **데이터 분석 및 시각화**: 파싱된 데이터는 사용자에게 의미 있는 정보(통계, 차트 등)로 시각화될 것입니다. 구글 맵 의존성으로 미루어 보아, 운동 경로를 지도 위에 표시하는 기능이 포함될 수 있습니다.
4.  **크로스플랫폼 지원**: Flutter의 장점을 활용하여, 한 번의 코드 베이스로 모바일, 웹, 데스크톱에서 일관된 사용자 경험을 제공하도록 설계되었습니다.

## 5. 심층 분석을 위한 제안
더 정확한 분석을 위해 다음 파일들의 내용을 직접 확인하는 것이 좋습니다.
1.  **`pubspec.yaml`**: 모든 외부 패키지 의존성을 확인합니다.
2.  **`lib/main.dart`**: 앱의 전체적인 구조, 라우팅, 상태 관리 초기화 로직을 파악합니다.
3.  **`screens/` 및 `widgets/`**: 데이터를 사용자에게 어떻게 보여주는지 UI 구조를 파악합니다.
4.  **`providers/`**: 어떤 상태 관리 솔루션(Provider, Riverpod, BLoC 등)을 사용했는지 확인합니다.
